# Elevator Pitch Generator

## When to use

- User asks for elevator pitch or personal intro
- User is preparing for networking event, job interview, investor meeting, or conference
- User wants a "tell me about yourself" answer

## Pattern

### 1. Gather context

Ask only what's missing — one question at a time:

- Current role and domain (specific: what problems, for whom)
- One concrete proof point — a number, result, or scale signal
- Context: networking / job interview / investor meeting / conference intro
- What they want next — a role, conversation, intro, or follow-up meeting

Skip questions if user has already provided context.

### 2. Select framework by context

| Context | Framework | Length |
|---|---|---|
| Networking event | Who I help → How → Why it matters → Hook question | 30–45s |
| Job interview | Present → Past → Future → close to the role | 60–90s |
| Investor meeting | Problem at scale → Solution → Traction → Explicit ask | 60s |
| Conference intro | Counterintuitive claim → what you're curious about here | 15–20s |

IMPORTANT: "What do you do?" and "Tell me about yourself" are different prompts. For casual networking, generate the short-form "what do you do" version first.

### 3. Rules for every version

- Open with frame, not name + title — frame before credentials
- Use one of three hook types:
  - Problem image: "I build the thing that breaks at 3am..."
  - Tension/reversal: "I spent X years doing Y and kept hitting Z, so I..."
  - Unusual claim: something that earns a follow-up question
- Use power verbs: built, shipped, fixed, led, cut, grew, decided, finished
- Avoid: "responsible for", "worked on", "helped with", "passionate about"
- Include exactly one number or named result
- End with curiosity opener, soft signal, or direct ask — never "so yeah, that's me"
- No jargon that only insiders recognize

### 4. Output

Produce three versions:

**30-second** — Problem → Solution → Ask
**60-second** — Problem → Solution → Proof → Ask
**Context-specific** — adapted to the chosen context from the table above

Annotate each with one sentence explaining the structural choice made.

IMPORTANT: Pitch must sound fluent, not scripted. Use contractions, allow one casual phrase, avoid relentlessly parallel structure.
